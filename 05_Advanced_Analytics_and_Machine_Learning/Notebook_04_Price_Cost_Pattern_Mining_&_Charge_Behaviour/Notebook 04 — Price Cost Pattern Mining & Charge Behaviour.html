)

# build a table at shipment level: which charges are present per consignment

# create a simple indicator: any line with this charge on the shipment
charge_presence = (
    df
    .groupby(["unique_tracking", "charge_type"])
    .size()
    .reset_index(name="lines")
)

charge_presence["has_charge"] = True

# correct â€” use df not df_ship
ship_core = df[[
    "unique_tracking",
    "client_code",
    "supplier_code",
    "service_level",
    "dest_country_code",
    "total_sales",
    "total_costs",
    "margin",
    "is_loss_making"
]].drop_duplicates("unique_tracking")

ship_with_charge = charge_presence.merge(ship_core, on="unique_tracking", how="left")

loss_by_charge = (
    ship_with_charge
    .groupby("charge_type")
    .agg(
        shipments=("unique_tracking", "nunique"),
        loss_shipments=("is_loss_making", "sum"),
        avg_margin=("margin", "mean"),
        avg_margin_pct=("margin_pct", "mean") if "margin_pct" in ship_core.columns else ("margin", "mean")
    )
    .reset_index()
)

loss_by_charge["loss_rate_%"] = np.where(
    loss_by_charge["shipments"] > 0,
    loss_by_charge["loss_shipments"] / loss_by_charge["shipments"] * 100,

)

loss_by_charge["loss_uplift_pp"] = loss_by_charge["loss_rate_%"] - baseline_loss_rate * 100

4. Logistic model: which charges increase loss risk?Â¶

We can go a step further and build a simple logistic regression that estimates the impact of each charge on the probability that a shipment is loss-making.

High-level idea:

  • For each shipment, engineer binary features like has_FuelSurcharge, has_ManualFee, etc.
  • Fit a logistic regression to predict is_loss_making.
  • Look at the coefficients to see which charges have the strongest positive / negative contribution to loss risk.

This is not a full causal model, but itâ€™s a very nice portfolio-level signal.
